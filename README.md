# 记录一次解决校园网环境下 Clash Verge 节点全部 Timeout 的过程

## 📌 问题描述

最近在使用 Clash Verge 作为代理工具时，遇到了一个在校园网环境下非常典型的网络连通性问题，具体表现如下：

1. **连接手机热点时**：Clash 可以正常使用，所有节点连通。
2. **切换到校园网时**：如果之前连着热点，刚切回校园网还能勉强用；但一旦断开重连，或者点击“更新订阅”后，Clash 就完全无法使用了。
3. **最终表现**：代理面板里的所有节点全部显示 `timeout`（超时）。

---

## 🔍 问题分析与原理探究

经过排查，这个问题的核心原因在于**校园网的 DNS 污染（域名劫持）**。

### 1. 为什么“切热点能用，切回校园网接着能用，更新后全挂”？
* **热点环境（干净网络）**：当你连接手机热点时，网络环境没有被劫持，Clash 成功将节点列表里的“域名”（例如 `hk.node.com`）解析成了真实的服务器“IP 地址”，并在本地建立了 DNS 缓存。
* **切回校园网（利用缓存）**：切回校园网后，Clash 直接使用了刚才缓存的真实 IP 地址进行通信，成功绕过了校园网的域名解析审查，所以看起来还能继续用。
* **更新订阅（缓存清空）**：当你点击更新订阅时，Clash 刷新了配置并清空了 DNS 缓存。此时，Clash 只能使用校园网的 DNS 重新去解析节点域名。校园网的防火墙拦截了这些请求，或者返回了错误的 IP，导致 Clash 找不到真实的服务器，从而全部显示 `timeout`。

### 2. 核心知识点：DNS 与 DoH
* **传统 DNS (明文传输)**：默认情况下，DNS 查询走的是 `UDP 53` 端口，且内容是纯明文的。校园网的网关（防火墙）非常容易识别并拦截、篡改这个端口的数据，这就是所谓的**DNS 劫持**。如果你在配置里写了 `8.8.8.8` 等常规 DNS，在校园网环境下不仅无效，还容易触发网络阻断。
* **DoH (DNS over HTTPS)**：这是一种通过 HTTPS 加密协议进行 DNS 查询的技术。它将 DNS 请求伪装成普通的网页浏览流量（走 443 端口），从而有效防止中间人（如校园网关）偷窥和篡改。
* **Fake-IP 模式**：Clash 的一种高效处理模式。开启后，当浏览器请求访问某个域名时，Clash 会立刻返回一个虚拟的假 IP（Fake-IP）给浏览器，让浏览器马上开始发送网络数据。真正的域名解析动作则被推迟并交给远端的代理服务器去完成，这对于突破本地网络限制非常有效。

---

## 🛠️ 最终解决方案

要彻底解决这个问题，需要修改 Clash Verge 的 DNS 配置，强制其使用国内的加密 DNS，并**严格分离“用于解析 DoH 域名的基础 IP”和“日常使用的加密 DNS 链接”**。

### 步骤一：打开 Clash Verge DNS 设置
在 Clash Verge 中进入 **设置 (Settings)** -> 找到 **DNS** 模块 -> 点击齿轮图标展开详细设置。
* 确保 **启用 (Enable)** 处于勾选状态。
* 确保 **模式 (Mode)** 设置为 `fake-ip`。

### 步骤二：修改默认域名服务器 (Default-nameservers)
这个配置的作用仅仅是用来解析“下面将要填写的加密 DNS（DoH）本身”的域名。因此，这里**只能填写纯 IP 地址**，推荐使用国内延迟较低的公共 IP。

将原有的内容替换为：
```text
223.5.5.5, 119.29.29.29
```
*(注：这里必须去掉 8.8.8.8 等容易被墙或被校园网重点劫持的海外 IP)*

### 步骤三：修改域名服务器 (Nameservers)
这个配置负责你所有日常上网的域名解析。为了防止校园网偷窥和篡改，这里**必须全部使用加密链接 (DoH)**，**绝对不能包含类似 `8.8.8.8` 这样的明文纯数字 IP 地址**（一旦包含明文 IP，流量一走 53 端口就会立刻再次被校园网劫持）。

将原有的内容替换为：
```text
https://dns.alidns.com/dns-query, https://doh.pub/dns-query
```

### 步骤四：重启生效与防范屏蔽
1. **保存并重启**：保存上述设置后，彻底退出 Clash Verge 并重新打开，使网络内核重新加载配置。
2. **通过代理更新 (进阶建议)**：为了防止机场订阅链接本身的域名被校园网直接屏蔽，建议在订阅设置中开启 **“通过代理更新 (Update via Proxy)”**。这样以后更新节点时，流量会走代理节点，彻底无视校园网的封锁。

---

## 💡 总结

在校园网或公司内网等具有严格网络审查的环境下使用代理工具，**防 DNS 污染**是第一要务。检查自己的配置中是否混入了明文的 `8.8.8.8` 等基础 DNS，全面拥抱 `DoH` 加密解析，搭配 `fake-ip` 模式，就能解决 99% 的连通性异常问题。
